# 复盘与协作经验

本文件沉淀历次任务的复盘与可迁移经验。**全局适用的精简原则**已提炼进根 [`AGENTS.md`](../../AGENTS.md) 的「协作效率原则」;此处保留完整叙事与推理,供需要时查阅——按需阅读,不占每次会话上下文。

## 复盘:Python → Rust Runtime + TS UI 语言/架构迁移(2026-07-06)

### 经过

一次"把项目从 Python 换语言"的任务,中途经历**两次方向掉头**:
1. Python → 纯 TypeScript(评估后:IO-bound、生态、部署形态 → 选 TS);
2. 纯 TS → **Rust Runtime + TS UI**(补充约束浮现:长运行 Runtime、本地+服务器双端、系统级隔离 → Runtime 定 Rust,UI 留 TS)。

最终落地:Cargo workspace 六层 crate + foundation/memory 契约、全文档 Rust 化、xtask 文档检查、覆盖率 93.43%,全链验证通过。

### 哪里费了 token / 返工(诚实记录)

| 返工点 | 浪费 | 根因 |
|---|---|---|
| 两次语言掉头 | 纯 TS 后端代码 + 14 篇文档的 TS 化**全部推倒重来**(翻两遍) | 语言决策前未先问清地基约束(部署形态/运行时寿命/系统级需求),第一次评估只覆盖了一部分维度 |
| 文档先 TS 化再 Rust 化 | 同批文档翻译两遍 | 方向未锁死就铺开大范围转换 |
| 子代理中途余额耗尽 | 2 个 agent 死掉、留半成品,需检测重派 | 未"先验证一个成功再批量扇出"(第二批才纠正) |
| cargo-llvm-cov 本地装不上 | 轮询等待十几分钟 | 重型/不确定安装未及早 time-box 并交用户 |
| model_gateway 命名 + 文档文件名 | 改名又改回(snake↔kebab 两轮) | 未先把"各类产物命名规范"一次定清(Rust 源码 snake vs 文档文件 kebab) |

### 做对的(保留)

- **大掉头前用结构化提问让用户拍板**——这是把返工成本控制住的关键,否则更贵。
- **第二批子代理"先验证一个成功再扇出"**——避免了整批浪费。
- **本地验证链(fmt/clippy/test)早跑**——问题当场暴露。
- **ADR 让设计资产跨语言零损失**——六层架构、事件协议、多租户不变量、memory 四件套全程未丢。

### 可迁移经验(→ 已提炼进根 AGENTS「协作效率原则」)

1. **地基决策先问全再动手**:架构/语言/存储/部署这类难逆转的选择,动手前一次性问清「部署形态·目标用户·运行时寿命·系统级需求」四类约束——一次问全会直接到终点,避免中途掉头推翻存量。
2. **方向未锁死前,不铺开可逆但昂贵的大范围转换**(全量改语言、全量改文档):等拍板再做,别做"很可能白费"的铺开。
3. **并行子代理:先确认一个成功,再批量扇出**;重型工具安装 time-box,失败早交用户,别死等。
4. **约定先立后用**:命名/风格规范一次定清再动手(Rust 源码 snake、文档文件 kebab、TS camelCase),避免改名往返。
5. **定向读取优于反复全量扫描**;复盘沉淀进本文件,不堆进每次载入的 AGENTS.md。

### 关于文档/记忆文件组织(本次采纳)

参照 Claude Code 官方 memory 最佳实践([support.claude.com](https://support.claude.com/en/articles/14553240-give-claude-context-claude-md-and-better-prompts)):根 `AGENTS.md` 每次会话全量载入、每行永久占 token,故只留**全局常用**;分场景细则放**子目录 `AGENTS.md`**(`crates/AGENTS.md`、`docs/AGENTS.md`),仅在该目录被触及时按需载入;完整复盘/参考材料放本文件,按需读取。单一事实源:命名等硬规则以 ADR 0014 为准,其余引用不重复维护。
